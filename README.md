## Almost Clinical: Linguistic Properties of Synthetic Electronic Health Records

This project evaluates how well large language models (LLMs) can generate synthetic mental‑health electronic health records (EHRs) that resemble real clinical writing. We outline the process for constructing a synthetic corpus and analyse how LLMs express agency, modality, and information flow across four key clinical genres: **Initial Assessments, GP Correspondence, Referrals, and Care Plans**.

While LLMs produce coherent, clinically styled text with appropriate terminology, they also show systematic limitations—such as shifts in register, reduced clinical specificity, and inaccuracies in medication and diagnostic details. These findings illustrate both the potential and the constraints of synthetic corpora as a privacy‑preserving resource for large‑scale linguistic research that cannot be conducted with genuine patient records.

### Genres Included

*   Initial Assessments (`Init`)
*   GP Correspondence (`GP`)
*   Referral / Handover Letters (`Ref`)
*   Care Plans (`Care`)

### Parameter Variations

Synthetic texts were generated by systematically varying:

*   **Age:** 25, 50
*   **Gender:** male, female
*   **Sexuality:** n/a, homosexual, bisexual
*   **Ethnicity:** variants of White British and Afro-Caribbean
*   **Diagnosis:** depressive disorders, bipolar disorders, cyclothymia
*   **Medication:** none or sertraline (100mg/200mg)
*   **Risks:** chronic pain, decreased libido, suicidal ideation, family history of suicide
*   **Treatment history:** no admissions, informal admissions, MHA Section 2, MHA Section 3

### Synthetic corpus
To create a list of clinical vignettes (stories) use:
```
python ./story-generator.py -t story-template.txt -j story-values.json -o generated-stories
```

In the output directory, this will produce a number of files with reasonable file names, each of which reflects a combination of story values with abbreviations for more than one word in any given value, for example, `25-female-bisexual-sedd-ts2mdotltm-wb-dl-ia.txt` 

To generate a corpus, combine a list of stories with the prompt. For example, to generate Care plans 3000 words max, use
```
python ./llama-generator.py -i story-list.txt -m HF/model.name -o llama-care.json -p care.txt -M 3000 -s system.prompt
```

This will output a json file. .I find it more convenient to work with markdown files produced by:
```
python ./extract-json.py FILE.json >FILE.md
```

This script also removes annoying boilerplate from LLMs.

### Corpus Structure
All generated texts are provided as JSON files, organised by clinical genre and by the LLM used to generate them.
Each file contains:

* input parameters (demographics, diagnosis, risks, etc.)
* the precise prompts used
* the resulting synthetic document text
